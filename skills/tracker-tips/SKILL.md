# Навык: Яндекс Трекер MCP — резолв аккаунтов и особенности

## Когда применять
Когда работаешь с Яндекс Трекером через MCP-инструменты и нужно найти пользователя или получить статистику.

## Резолв пользователей

### resolve_user
Работает по **русскому имени**, не по кратким алиасам:
- ✅ "Иванов", "Сергей Иванов"
- ❌ "IvanovS", "ivanov" (краткие латинские алиасы)

### get_employee_stats
Требует **старый логин** (Яндекс паспорт) ИЛИ полное русское имя:
- Новые логины (после миграции) не работают
- Если инструмент не находит — попробовать полное ФИО на русском

### search_issues
Требует **новый логин**:
- Старые логины работают только для задач, назначенных на старый аккаунт

## Маппинг логинов (пример)

У каждого сотрудника может быть ДО ТРЁХ разных идентификаторов:

| Тип | Пример | Где используется |
|-----|--------|-----------------|
| Старый логин | `username123` | get_employee_stats |
| Новый логин | `ivanov` | search_issues |
| Трекер-логин | может отличаться (!) | Некоторые сотрудники зарегистрированы под неочевидным логином |

**Составьте свой маппинг** через `resolve_user` по ФИО каждого сотрудника.

## Важно
- **Ворклоги** могут быть 0 если команда не логирует время
- **DevOps/фронтенд** сотрудники часто не ведут задачи в Трекере
- При объединении данных из разных инструментов — проверяйте какой формат логина нужен каждому
