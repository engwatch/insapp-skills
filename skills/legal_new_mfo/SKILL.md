# /legal_new_mfo — Договоры лидогенерации МФО

## Что делает скилл

1. Добавляет нового партнёра в скрипт `make_all_contracts.py`
2. Запускает скрипт — генерирует `.docx` договоры в папку `Готовые договора МФО`
3. Обновляет реестр договоров через MCP `gsheets_batch_update`
4. **Google Drive** — пользователь загружает сам

---

## Ключевые пути и настройки

| Параметр | Значение |
|---|---|
| Скрипт генерации | `legal/make_all_contracts.py` |
| Выходная папка | `legal/Готовые договора МФО/` |
| Реестр spreadsheet ID | `YOUR_REGISTRY_SPREADSHEET_ID` |
| Лист реестра | `договоры` (настраивается) |

## Колонки реестра (gsheets_batch_update)

Уточни у пользователя какие колонки заполнять. Типовые:

| Колонка | Содержимое |
|---|---|
| A | Порядковый № (цифра) |
| B | Юрлицо с организационной формой (ООО «…» / ИП Фамилия И.О.) |
| D | Бренд партнёра |
| H | Номер договора (`07-03/2026-МФО`) |

Колонки с дропдаун-валидацией **не трогать** — проверь перед записью.

---

## Нумерация договоров

- Формат номера: `{NN}-{MM}/{YYYY}-МФО` — пример: `07-03/2026-МФО`
- Формат имени файла: `{NN}-{MM}.{YYYY}-МФО_{brand}_Договор_лидогенерации.docx` (`.` вместо `/`)

**Как найти следующий номер:** прочитать реестр (колонка A), взять максимальное число + 1.

```python
# В скрипте:
contract_num = f'{num:02d}-03/2026-МФО'
filename = f'{num:02d}-03.2026-МФО_{brand}_Договор_лидогенерации.docx'
```

## Структура партнёра (PARTNERS list)

```python
{
    'brand': 'Название бренда',           # для имени файла и реестра
    'legal': 'ООО «ЮРЛИЦО»',             # полное юр. название
    'inn': '1234567890',
    'kpp': '123456789',                   # '' для ИП
    'ogrn': '1234567890123',              # ОГРН или ОГРНИП (15 цифр для ИП)
    'address': '123456, г. Москва, ...',
    'bank': 'Название банка',
    'ks': '30101810...',
    'bik': '044525...',
    'rs': '40702810...',
    'title': 'Генеральный директор',      # или 'Индивидуальный предприниматель'
    'fio': 'И.О. Фамилия',
    'percent': '70',                      # % вознаграждения исполнителя, '___' если неизвестно
}
```

**Особенности ИП:**
- `kpp` = `''`
- `ogrn` — 15-значный ОГРНИП
- `title` = `'Индивидуальный предприниматель'`
- `legal` = `'ИП Фамилия И.О.'`
- в договоре автоматически используется `ИНН` (без КПП) и `ОГРНИП`

## Алгоритм выполнения

### Шаг 1 — Определить номера

Прочитать реестр через MCP:
```
gsheets_read(spreadsheet_id="YOUR_REGISTRY_SPREADSHEET_ID", range="договоры!A:A")
```
Найти последнюю заполненную строку и её номер → `START_NUM = max_num + 1`.

### Шаг 2 — Добавить партнёров в скрипт

Открыть `make_all_contracts.py`, добавить в `PARTNERS`, обновить стартовый номер и текущую дату.

### Шаг 3 — Запустить скрипт

```bash
python3 make_all_contracts.py
```

### Шаг 4 — Обновить реестр через MCP

```python
# Пример для 2 новых партнёров, строки 20-21
ranges = [
    "договоры!A20", "договоры!B20", "договоры!D20", "договоры!H20",
    "договоры!A21", "договоры!B21", "договоры!D21", "договоры!H21",
]
values = [
    [["15"]], [["ООО «НОВЫЙ ПАРТНЁР»"]], [["Бренд1"]], [["15-03/2026-МФО"]],
    [["16"]], [["ООО «ДРУГОЙ ПАРТНЁР»"]], [["Бренд2"]], [["16-03/2026-МФО"]],
]
```

```
gsheets_batch_update(spreadsheet_id="YOUR_REGISTRY_SPREADSHEET_ID",
                     ranges=ranges, values=values)
```

### Шаг 5 — Сообщить пользователю

Список созданных файлов + путь к папке. Напомнить загрузить в Google Drive вручную.

## Частые ошибки

| Ошибка | Решение |
|---|---|
| MCP не работает с `.xlsx` файлами | Использовать только нативный Google Sheets (URL `/spreadsheets/d/...`) |
| Запись в колонку с дропдауном → ошибка валидации | Пропустить эту колонку |
| Playwright автозаполняет бренд | Использовать только MCP для записи в таблицу |
| Договор выглядит как «Франкенштейн» | Структура должна копировать уже существующий договор-образец лидогенерации, а не гибридироваться |
