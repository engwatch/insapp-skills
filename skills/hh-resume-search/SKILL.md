# Навык: Эффективный поиск кандидатов на hh.ru через Playwright

## Когда применять
Когда нужно найти специалистов узкого профиля на hh.ru через Playwright.

## Что НЕ работает
- Длинные запросы с кучей слов ("CPA affiliate арбитраж трафика МФО") — hh.ru матчит каждое слово отдельно, выдаёт миллионы нерелевантных результатов
- OR, кавычки, точные фразы — hh.ru их игнорирует
- URL-based поиск с фильтрами (experience=between3And6 и т.д.) — hh.ru возвращает fallback/рекомендуемые резюме (одни и те же для ЛЮБОГО запроса)

## Что РАБОТАЕТ

### 1. Form-based поиск (основной метод)
```
page.goto('/search/resume') → fill input → Enter
```
Или добавить параметр в URL: `hhtmFrom=resume_search_result&hhtmFromLabel=resume_search_line`

### 2. Короткие точные запросы (2-3 слова)
- "affiliate manager финансы" — 609 результатов, релевантные
- "CPA сеть МФО PDL" — 12 результатов, точные
- "Банки.ру affiliate партнёрская программа" — 3 результата, идеальные
- Названия конкретных компаний (Click2Money, LeadGid, Fincpanetwork) — самые точные

### 3. Ввод через Playwright
- Click на поле поиска
- Meta+a (выделить всё)
- Type новый запрос с submit=true

### 4. Блок "Похожие резюме" — ЗОЛОТАЯ ЖИЛА
Внизу каждого резюме — ML-матчинг по навыкам и опыту. Находит то, что текстовый поиск пропускает. Через него найдены золотые кандидаты.

### 5. Комбинации "роль + вертикаль"
- "affiliate manager финансы"
- "head of affiliate финансовая вертикаль"
- "CPA сеть МФО PDL"

## Фильтры
- `experience=between3And6&experience=moreThan6` — только опытные (НО ломает URL-поиск!)
- `schedule=remote` — часто отсекает хороших, лучше убирать
- `pos=full_text` — поиск по всему тексту резюме

## Ограничения hh.ru
- hh.ru подменяет специфические названия ("Saleads" → "Sales", "LeadGid" → "leaded")
- Сотрудники маленьких компаний (15 чел.) практически не публикуют резюме
- Google НЕ индексирует отдельные резюме hh.ru

## Почему это работает
hh.ru ищет по токенам с релевантностью. Короткие запросы из 2-3 слов дают более точные результаты, чем длинные из 5+. Блок "похожих резюме" использует ML-матчинг — находит то, что текстовый поиск пропускает.
