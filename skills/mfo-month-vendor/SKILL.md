# /mfo-month-vendor — Сверка МФО по вендорам за месяц

## Что делает

Создаёт Google Spreadsheet со сверкой по вендорам за указанный месяц:
- Листы по каждому партнёру (Локо, Долинск и т.д.) со строками сделок
- Лист Итог с формулами: получаем от вендоров, исходящее КВ, остаток у Инсап, расшифровка по вендорам

## Входные данные

При вызове уточни у пользователя (если не указано):
- Период (месяц/год)
- Партнёры и их % (Локо = 100%, Долинск = 70%)
- Файлы вендоров и правила назначения партнёру (какие кампании → какой партнёр)
- Инвестиции Инсап (если есть): даты и суммы для Локо

## Алгоритм выполнения

### Шаг 1. Подготовка данных

1. Прочитать вендорские файлы (Drive или локально)
2. По каждой строке определить партнёра (по кампании WB, по правилу из условия)
3. Добавить колонку H = "Вендор" (Финуслуги/Финбридж/Манимен/Веббанкир)
4. Для Локо добавить строки Инвестиций Инсап **в хронологическом порядке** (не в конец!)

### Шаг 2. Создание таблицы

Скопировать шаблон:
```
drive_copy_file(fileId="1M0EImzWTNc916nhGs3ygBjYgB63Da055UMZl77QKkA8", name="Сверка [месяц] [год]")
```

### Шаг 3. Заполнение листов партнёров

Структура колонок:
| Col | Название | Формула |
|-----|----------|---------|
| A | CREATE_TIMESTAMP_AP | дата конверсии |
| B | CREATE_TIMESTAMP_LEAD | дата+время лида |
| C | ID_LEAD | click-id |
| D | NAME_OF_PARTNER | имя МФО из файла вендора |
| E | Инсап входящее КВ | сумма от вендора |
| F | Процент деления прибыли | 100% или 70% |
| G | КВ для партнёра | =E*F |
| H | Вендор | метка вендора |

Использовать `gsheets_batch_update` с `valueInputOption: "USER_ENTERED"`.

**⚠ Десятичный разделитель:** запятая `,` (не точка). `=E2*0,7` — правильно.

### Шаг 4. Заполнение листа Итог

```
B1  = =B2
B2  = =SUM(B16:E16)      ← Получаем от вендоров
B4  = =B5+B7             ← Исходящее КВ
B5  = =SUM('Локо'!G2:G1000)
B6  = =SUMPRODUCT(('Локо'!H2:H100="Инвестиции Инсап")*('Локо'!G2:G100))
B7  = =SUM('Долинск'!G2:G1000)
B9  = =B2-B4             ← Остаётся у Инсап

Строка 13: заголовки — B=ФУ, C=Финбридж, D=Манимен, E=Веббанкир
Строка 14: ЛОКО
Строка 15: Долинск
Строка 16: Итого
```

**Формулы строк 14–15 — КРИТИЧЕСКИ ВАЖНО:**
Каждая ячейка = SUMPRODUCT **только по своему листу**:
```
B14 = =SUMPRODUCT(('Локо'!H2:H100="Финуслуги")*('Локо'!E2:E100))
C14 = =SUMPRODUCT(('Локо'!H2:H100="Финбридж")*('Локо'!E2:E100))
D14 = =SUMPRODUCT(('Локо'!H2:H100="Манимен")*('Локо'!E2:E100))
E14 = =SUMPRODUCT(('Локо'!H2:H100="Веббанкир")*('Локо'!E2:E100))

B15 = =SUMPRODUCT(('Долинск'!H2:H100="Финуслуги")*('Долинск'!E2:E100))
C15 = =SUMPRODUCT(('Долинск'!H2:H100="Финбридж")*('Долинск'!E2:E100))
D15 = =SUMPRODUCT(('Долинск'!H2:H100="Манимен")*('Долинск'!E2:E100))
E15 = =SUMPRODUCT(('Долинск'!H2:H100="Веббанкир")*('Долинск'!E2:E100))

B16 = =B14+B15   C16 = =C14+C15   D16 = =D14+D15   E16 = =E14+E15
```

### Шаг 5. Форматирование

1. Заголовки: жирный шрифт, цветная заливка
2. Числовые колонки E/G: числовой формат
3. Ширина колонок: Playwright auto-fit (скилл `column-auto-width`)
4. Преобразовать в таблицу: Playwright (скилл `convert-to-table`)

## Обязательные проверки (тройная верификация)

### После шага 3 (данные в листах):
- [ ] Прочитать SUM(E), SUM(G) каждого листа
- [ ] Убедиться что SUM(G) ≈ SUM(E) × (% партнёра)
- [ ] Проверить количество строк: совпадает ли с источниками
- [ ] Убедиться что нет дублей и нет пустых строк в конце

### После шага 4 (формулы Итог):
- [ ] Прочитать B2, B4, B5, B7, B9
- [ ] Проверить: B2 = B5 − инвестиции + SUM(Долинск!E) / 0.7 × некорректно... просто убедиться что B2 ≈ B4 (небольшая разница = маржа Инсап)
- [ ] Проверить breakdown: сумма B16+C16+D16+E16 = B2
- [ ] Убедиться что у Локо нет "Манимен" в breakdown (D14 должно быть 0 если у Локо нет Манимен-вендора)
- [ ] Убедиться что итоги строк 16 = суммам колонок 14+15

### Контрольное вычисление:
```
Ожидаемый B5 = SUM всех G в Локо
Ожидаемый B7 = SUM всех G в Долинск
Ожидаемый B4 = B5 + B7
Ожидаемый B2 = SUM(B16:E16) = сумма E по всем вендорам обоих листов (искл. инвестиции)
```

## Частые ошибки

### gsheets_read смещает строки
`gsheets_read('Локо!A2:H80')` → строка A2 уходит в `columnHeaders`, данные начинаются с A3. При Python-обработке: явно включать headers обратно, или читать с A1.

### Перезапись диапазона не чистит хвост
Если писали A2:H79 (78 строк), но потом данных стало 77 — строка 79 останется старой. После перезаписи явно делать `gsheets_delete_rows` для лишних строк.

### Смешивание листов в SUMPRODUCT breakdown
Каждая ячейка таблицы breakdown = **один лист**. Если суммировать два листа в одной ячейке — получится итог вместо строки партнёра, данные задвоятся в строке Итого.

### Формулы в G-колонке
Писать `=E2*F2` (не hardcode). Десятичный в формулах — запятая: `=E2*0,7`.

### Порядок вставки инвестиций
Строки "Инвестиции Инсап" вставлять по дате лида (B-колонка), не в конец таблицы. Сортировка — по убыванию даты (новые вверху).
