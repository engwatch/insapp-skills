# /mfo-month-vendor — Сверка МФО по вендорам за месяц

## Что делает

Создаёт Google Spreadsheet со сверкой по вендорам за указанный месяц:
- Листы по каждому партнёру (Локо, Долинск) со строками сделок
- Лист **Инсап** — собственный трафик на ключе МФО Инсап (НЕ исключать!)
- Лист **Итог** с формулами: получаем от вендоров, исходящее КВ, собственный доход Инсап, остаток у Инсап, расшифровка по вендорам

## Входные данные

При вызове уточни у пользователя (если не указано):
- Период (месяц/год)
- Партнёры и их % (Локо = 100%, Долинск = 70%) — % берутся из БД, но Локо всегда override 100%
- Папка с файлами вендоров
- Инвестиции Инсап (если есть): даты и суммы для Локо

## Алгоритм выполнения

### Шаг 1. Анализ вендорских файлов

1. Прочитать все xlsx из указанной папки через `openpyxl`
2. Определить структуру каждого файла: какие колонки содержат дату, ID, сумму, offer_id
3. Извлечь offer_id / click_id для привязки к БД:
   - **Финуслуги**: External ID (col 9) = ApplicationId, Token 2 (col 14) = OfferId
   - **Деньги Сразу**: Основной внешний параметр (col 9) = OfferId
   - **One Click Money**: URL (col 4) содержит `offer_id=...` — извлечь regex. **Если offer_id = `[[click_id]]`** (шаблон не заменён) — использовать Id Заявки (col 1) как fallback
   - **Быстроденьги**: aff_sub1 (col 10) = OfferId
   - **Другие вендоры**: искать GUID-подобные поля, проверять через БД

### Шаг 2. Определение партнёра через БД (ОБЯЗАТЕЛЬНО)

**НЕ УГАДЫВАТЬ партнёра по названию кампании. ВСЕГДА проверять через БД.**

Цепочка: `FinOffers.OfferId → FinOffers.ApplicationId → Applications.ApiKeyId → PartnerApiKeys.PartnerId → Partners.Name`

```sql
SELECT fo.OfferId, fo.IncomingComissionAmount, fo.OutgoingComissionAmount,
       fg.Name as FinOrgName, p.Name as PartnerName, p.PartnerId
FROM FinOffers fo
JOIN Applications a ON fo.ApplicationId = a.ApplicationId
JOIN PartnerApiKeys pak ON a.ApiKeyId = pak.ApiKeyId
JOIN Partners p ON pak.PartnerId = p.PartnerId
JOIN FinOrgs fg ON fo.FinOrgId = fg.FinOrgId
WHERE fo.OfferId IN (...)
```

**Известные партнёры и их PartnerId:**

| Партнёр | PartnerId | Сплит |
|---------|-----------|-------|
| КБ "ЛОКО-БАНК" (АО) | 4e96325d-0734-4989-bddd-eebf459d132e | 100% (override) |
| КБ "ДОЛИНСК" (АО) | a13932e8-85db-4862-be94-1de935f8c81c | 70% (из БД) |
| МФО Инсап | a5842f98-c8a8-4a91-9dec-7cd27e734c00 | 100% (собственный) |

**КРИТИЧНО:** Офферы на ключе МФО Инсап (OutgoingComission = 0) — это собственный доход Инсап. Они идут на отдельный лист «Инсап», а НЕ исключаются из сверки.

### Шаг 3. Создание таблицы

Попробовать скопировать шаблон:
```
drive_copy_file(fileId="1M0EImzWTNc916nhGs3ygBjYgB63Da055UMZl77QKkA8", name="Сверка [месяц] [год]")
```

**Если нет прав на копирование** — создать с нуля:
```
gsheets_create_spreadsheet(title="Сверка [месяц] [год]", sheets=[{title:"Итог"}, {title:"Локо"}, {title:"Инсап"}, {title:"Долинск"}])
```

### Шаг 4. Подготовка данных (Python)

1. Для каждой строки каждого вендорского файла:
   - Извлечь offer_id → определить партнёра через БД
   - Маппинг: Локо / Долинск / Инсап
   - Присвоить метку вендора для колонки H
2. Сортировка по дате (новые вверху)
3. Для Локо добавить строки Инвестиций Инсап **в хронологическом порядке** (не в конец!)

**⚠ Unicode-артефакты:** При чтении xlsx через openpyxl кириллические строки могут содержать битые символы (напр. `Ден��гиСразу`). После подготовки данных — **проверить все строки на наличие `\ufffd` или `?`** и заменить на корректные значения.

### Шаг 5. Заполнение листов партнёров

Структура колонок (одинаковая для всех листов — Локо, Долинск, Инсап):
| Col | Название | Содержимое |
|-----|----------|-----------|
| A | CREATE_TIMESTAMP_AP | дата конверсии |
| B | CREATE_TIMESTAMP_LEAD | дата+время лида |
| C | ID_LEAD | offer_id (или app_id если offer_id недоступен) |
| D | NAME_OF_PARTNER | имя МФО из файла вендора |
| E | Инсап входящее КВ | сумма от вендора (из файла, НЕ из БД) |
| F | Процент деления прибыли | 100% или 70% |
| G | КВ для партнёра | =E*F |
| H | Вендор | метка вендора (Финуслуги / Деньги Сразу / One Click Money / Быстроденьги и т.д.) |

**ВАЖНО:** Использовать **только** `gsheets_batch_update` с `valueInputOption: "USER_ENTERED"`.
Инструмент `gsheets_update_cell` НЕ поддерживает USER_ENTERED — формулы записываются как текст!

**⚠ Десятичный разделитель:** запятая `,` (не точка). Числа с дробной частью передавать как строку `"10137,25"`.

### Шаг 6. Заполнение листа Итог

**Вендоры динамические** — не хардкодить ФУ/Финбридж/Манимен/Веббанкир. Использовать вендоров из текущего месяца.

Структура (N = количество вендоров, M = строка Итого в breakdown):

```
Строки 1–9: Сводка
  A1 = "Входящее КВ итого"          B1 = =B2
  A2 = "Получаем от вендоров за [месяц]"  B2 = =SUM(B[M]:?[M])  ← строка Итого в breakdown
  A3 = ""
  A4 = "Исходящее КВ"               B4 = =B5+B7
  A5 = "Выплачиваем Локо"           B5 = =SUM('Локо'!G2:G1000)
  A6 = "Собственный доход Инсап"    B6 = =SUM('Инсап'!E2:E1000)
  A7 = "Выплачиваем Долинск"        B7 = =SUM('Долинск'!G2:G1000)
  A8 = ""
  A9 = "Остаётся у Инсап"           B9 = =B2-B4

Строка 13: заголовки вендоров  — "", [Вендор1], [Вендор2], ...
Строка 14: ЛОКО               — SUMPRODUCT по листу Локо для каждого вендора
Строка 15: Долинск             — SUMPRODUCT по листу Долинск
Строка 16: Инсап               — SUMPRODUCT по листу Инсап
Строка 17: Итого               — =B14+B15+B16, =C14+C15+C16, ...
```

**⚠ КРИТИЧНО — номер строки Итого:** B2 = `=SUM(B17:?17)` — ссылаться на **фактическую** строку Итого (17), а НЕ на строку ниже (18). Пересчитать после записи!

**Формулы SUMPRODUCT — каждая ячейка = один лист, один вендор:**
```
=SUMPRODUCT(('Локо'!H2:H100="Деньги Сразу")*('Локо'!E2:E100))
```

### Шаг 7. Форматирование

1. Заголовки всех листов: жирный, тёмный фон (#334D66), белый текст
2. Числовые колонки E/G: формат `#,##0.00`
3. Строка «Остаётся у Инсап»: зелёный фон, жирный, fontSize 12
4. Строка «Итого» в breakdown: серо-голубой фон, жирный
5. Ширина колонок: `gsheets_auto_resize` (НЕ Playwright)

### Шаг 8. Форматирование через Playwright (опционально)

Если пользователь просит:
- Преобразовать в таблицу: скилл `convert-to-table`
- Auto-fit: скилл `column-auto-width`

## Обязательные проверки (тройная верификация)

### После шага 5 (данные в листах):
- [ ] `gsheets_read` SUM(E), SUM(G) каждого листа
- [ ] SUM(G) ≈ SUM(E) × (% партнёра) — для Локо равны, для Долинск 70%
- [ ] Количество строк совпадает с источниками
- [ ] Нет дублей, нет пустых строк, нет `[[` или `\ufffd` в данных

### После шага 6 (формулы Итог):
- [ ] Прочитать B1, B2, B4, B5, B6, B7, B9
- [ ] B2 = SUM всех вендоров по ВСЕМ партнёрам (Локо + Долинск + Инсап)
- [ ] B4 = B5 + B7 (только внешние партнёры, без Инсап)
- [ ] B6 = собственный доход Инсап
- [ ] B9 = B2 − B4 (включает и маржу от внешних, и весь доход Инсап)
- [ ] Сумма строки Итого в breakdown = B2
- [ ] Каждая ячейка breakdown ссылается ТОЛЬКО на свой лист

### Контрольное вычисление:
```
Ожидаемый B5 = SUM всех G в Локо
Ожидаемый B7 = SUM всех G в Долинск
Ожидаемый B6 = SUM всех E в Инсап
Ожидаемый B4 = B5 + B7
Ожидаемый B2 = SUM всех E по всем листам (Локо + Долинск + Инсап, искл. инвестиции)
Ожидаемый B9 = B2 - B4
```

## Частые ошибки

### 1. Исключение трафика МФО Инсап
**НЕПРАВИЛЬНО:** исключить офферы на ключе МФО Инсап из сверки.
**ПРАВИЛЬНО:** собственный трафик Инсап = отдельный лист «Инсап», учитывается в Итоге как доход компании (строка B6 + строка в breakdown).

### 2. gsheets_update_cell для формул
**НЕПРАВИЛЬНО:** `gsheets_update_cell(value="=SUM(B17:E17)")` — запишет как текст.
**ПРАВИЛЬНО:** всегда `gsheets_batch_update` с `valueInputOption: "USER_ENTERED"`.

### 3. Unicode-артефакты из xlsx
При передаче кириллических строк через JSON/MCP могут появляться символы `�`. После записи данных — визуально проверить колонку D через `gsheets_read`. Битые ячейки перезаписать вручную через `gsheets_batch_update`.

### 4. [[click_id]] плейсхолдеры
В файлах One Click Money URL может содержать `offer_id=[[click_id]]` (шаблон не подставлен). Проверять после regex-извлечения. Если `[[` в значении — использовать Id Заявки из col 1 как fallback.

### 5. Номер строки Итого в формуле B2
Формула B2 должна ссылаться на **фактическую** строку «Итого» в breakdown. Если в breakdown 3 партнёра (строки 14-16) + Итого (строка 17) → B2 = `=SUM(B17:E17)`, а НЕ `B18:E18`. Всегда пересчитывать: 13 (заголовки) + кол-во партнёров + 1 (Итого).

### 6. gsheets_read смещает строки
`gsheets_read('Локо!A2:H80')` → строка A2 уходит в `columnHeaders`, данные начинаются с A3. При Python-обработке: явно включать headers обратно, или читать с A1.

### 7. Перезапись диапазона не чистит хвост
Если писали A2:H79, но потом данных стало 77 — строка 79 останется старой. После перезаписи явно делать `gsheets_delete_rows`.

### 8. Смешивание листов в SUMPRODUCT breakdown
Каждая ячейка = **один лист, один вендор**. Никогда не суммировать два листа в одной ячейке.

### 9. Десятичный разделитель и разделитель аргументов
Русская локаль Google Sheets: десятичный = `,`, разделитель аргументов = `;`.

### 10. Вендоры меняются от месяца к месяцу
НЕ хардкодить список вендоров (ФУ/Финбридж/Манимен/Веббанкир). Каждый месяц состав может быть другим (Деньги Сразу, One Click Money, Быстроденьги и т.д.). Определять по фактическим файлам.

### 11. Права на копирование шаблона
`drive_copy_file` может вернуть Insufficient Permission. Fallback: `gsheets_create_spreadsheet` + `gsheets_add_sheet`.
