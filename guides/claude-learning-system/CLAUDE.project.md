## Самопроверка при работе с данными

- После записи данных (БД, API, таблицы) — ОБЯЗАТЕЛЬНО прочитать обратно и сверить с ожидаемыми значениями.
- После записи диапазона — проверить что "хвост" (за пределами нового диапазона) не содержит старых данных.
- Форматирование/оформление — ТОЛЬКО после подтверждения корректности данных, НИКОГДА в одной операции с записью.

## Работа с внешними сервисами

- Каждая загрузка/перемещение файла = проверка что файл реально появился в месте назначения.
- При батч-загрузке (>10 файлов): после каждых 10 файлов — проверка count в папке назначения.
- Если 3 операции подряд не удались — СТОП, диагностика, отчёт пользователю.
