# Claude Code Learning System

Система самообучения Claude Code — журналирование, извлечение навыков и долгосрочная память между сессиями.

## Проблема

Claude Code по умолчанию:
- Не помнит предыдущие сессии
- Повторяет одни и те же ошибки
- Не учится на обратной связи пользователя
- Теряет контекст при длинных задачах (context window compaction)

## Решение: три уровня памяти

```
┌─────────────────────────────────────────────────────┐
│  Уровень 1: Журнал задачи (project/log.md)          │
│  Живёт: пока задача актуальна                       │
│  Создаётся: каждую сессию автоматически              │
├─────────────────────────────────────────────────────┤
│  Уровень 2: Навыки (~/.claude/skills/)              │
│  Живёт: постоянно                                   │
│  Создаётся: когда найден рабочий приём              │
├─────────────────────────────────────────────────────┤
│  Уровень 3: Память (memory/*.md + MEMORY.md)        │
│  Живёт: постоянно, загружается каждую сессию        │
│  Создаётся: при коррекциях, новых фактах, паттернах │
└─────────────────────────────────────────────────────┘
```

### Уровень 1: Журнал задачи

Каждая сессия создаёт папку `project/ГГГГ-ММ-ДД-название/log.md` с:
- Планом работы
- Прогрессом по шагам (с конкретными данными, не «сделано»)
- Обнаруженными навыками
- Итогом и артефактами

**Зачем:** другая сессия Claude может прочитать лог и продолжить работу с того места, где остановились. Защита от потери контекста при compaction.

### Уровень 2: Навыки (Skills)

Когда Claude находит рабочий приём (потратил >2 попыток, нашёл неочевидный параметр API, обходной путь) — **сразу** создаёт скилл в `~/.claude/skills/категория/SKILL.md`.

**Зачем:** следующая сессия не будет изобретать велосипед. Claude проверяет существующие навыки перед решением задачи.

### Уровень 3: Память

Файлы `memory/*.md` с индексом `MEMORY.md` (загружается каждую сессию). Типы:
- **feedback** — как пользователь хочет работать (коррекции и одобрения)
- **project** — факты о проекте (люди, ID, маппинги, алиасы)
- **user** — профиль пользователя (роль, экспертиза, цели)
- **reference** — где искать информацию (внешние системы, дашборды)

**Зачем:** Claude адаптируется к стилю работы пользователя и не переспрашивает известные вещи.

## Установка

### 1. Скопируй CLAUDE.md

```bash
# Глобальные правила (для всех проектов)
cp claude-learning-system/CLAUDE.global.md ~/.claude/CLAUDE.md

# Правила проекта (в корень рабочей директории)
cp claude-learning-system/CLAUDE.project.md /path/to/your/project/CLAUDE.md
```

### 2. Создай папку проектов

```bash
mkdir -p /path/to/your/project/project
```

### 3. Инициализируй память

```bash
# Claude Code создаст memory/ автоматически при первом сохранении
# Но можно создать MEMORY.md заранее:
mkdir -p ~/.claude/projects/-path-to-project/memory
echo "# Memory" > ~/.claude/projects/-path-to-project/memory/MEMORY.md
```

> **Путь memory/**: Claude Code хранит память по проекту в `~/.claude/projects/<encoded-path>/memory/`. Путь кодируется заменой `/` на `-`. Например, для проекта в `/Users/john/work/myapp` путь будет `~/.claude/projects/-Users-john-work-myapp/memory/`.

### 4. (Опционально) Установи cc — автоименование сессий

```bash
curl -o ~/.local/bin/cc https://raw.githubusercontent.com/engwatch/insapp-skills/main/tools/cc/cc
chmod +x ~/.local/bin/cc

# Для zsh — обязательно noglob:
echo 'alias cc='\''noglob ~/.local/bin/cc'\''' >> ~/.zshrc
source ~/.zshrc
```

## Как это работает на практике

### День 1: Первая сессия
```
Пользователь: сделай отчёт по продажам за март

Claude:
1. Создаёт project/2026-03-23-sales-report/log.md
2. Пишет план, выполняет шаги, записывает результаты
3. Обнаруживает что API возвращает даты в UTC → создаёт скилл
4. Пользователь говорит "не используй тёмную тему" → сохраняет в feedback-память
5. Завершает: обновляет статус, записывает итог
```

### День 2: Следующая сессия
```
Пользователь: сделай такой же отчёт за апрель

Claude:
1. Читает memory/ → знает про светлую тему
2. Проверяет skills/ → находит навык про UTC-даты
3. Читает project/2026-03-23-sales-report/log.md → понимает формат
4. Делает отчёт быстрее и без ошибок
```

### День 5: "Вспомни"
```
Пользователь: вспомни тот отчёт что мы делали на прошлой неделе

Claude:
1. ls project/ → находит 2026-03-23-sales-report
2. Читает log.md → понимает контекст
3. Продолжает или повторяет с нужными изменениями
```

## Структура файлов

```
~/.claude/
├── CLAUDE.md                    # Глобальные правила
├── skills/
│   ├── api-dates-utc/
│   │   └── SKILL.md             # Навык: работа с UTC-датами
│   └── sheets-formatting/
│       └── SKILL.md             # Навык: форматирование таблиц
└── projects/
    └── -path-to-project/
        └── memory/
            ├── MEMORY.md        # Индекс (загружается каждую сессию)
            ├── feedback_*.md    # Коррекции пользователя
            ├── project_*.md     # Факты о проекте
            └── user_*.md        # Профиль пользователя

/path/to/project/
├── CLAUDE.md                    # Правила проекта
└── project/
    ├── 2026-03-23-sales-report/
    │   ├── log.md               # Журнал задачи
    │   └── skill-*.md           # Навыки (до промоутинга)
    └── 2026-03-24-api-fix/
        └── log.md
```

## Ключевые принципы

### 1. Верификация после каждого шага
Не считай шаг выполненным без проверки. "Команда не выдала ошибку" ≠ "всё сработало".

### 2. Навыки создаются СРАЗУ
Не жди конца сессии. Нашёл рабочий приём → сразу в `~/.claude/skills/`. Сессия может оборваться в любой момент.

### 3. Память пишется ПО ХОДУ работы
Триггеры: пользователь поправил, одобрил неочевидное, узнал новый факт. Не жди конца — сохраняй сразу.

### 4. Логи для другой сессии
Лог должен быть достаточно подробным, чтобы ДРУГАЯ сессия Claude могла прочитать и продолжить. Не "выполнено 15 запросов", а конкретные цифры и выводы.

### 5. Перечитывай log.md каждые 5 шагов
Context compaction стирает детали. Периодическое перечитывание лога защищает от потери контекста.

## FAQ

**Q: Не перегрузится ли память?**
MEMORY.md — индекс, ~150 строк. Загружается каждую сессию (~2-3K токенов из 1M). Детали в отдельных файлах — загружаются по необходимости.

**Q: Сколько токенов тратится на журналирование?**
~5-10% от сессии. Окупается тем, что следующая сессия не повторяет ошибки.

**Q: Можно ли использовать с Cursor / другими IDE?**
Да. CLAUDE.md и skills/ работают в любом окружении где запускается Claude Code. Память привязана к пути проекта.

**Q: Что если Claude создаёт слишком много памяти?**
В CLAUDE.md есть правило: "не сохраняй очевидное или одноразовое". Периодически ревьюьте memory/ и удаляйте устаревшее.
