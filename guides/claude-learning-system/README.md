# Claude Code Learning System v2.0

Система самообучения Claude Code — СТОП-протокол, журналирование, извлечение навыков и долгосрочная память между сессиями.

## Проблема

Claude Code по умолчанию:
- Не помнит предыдущие сессии
- Повторяет одни и те же ошибки
- Не учится на обратной связи пользователя
- Теряет контекст при длинных задачах (context window compaction)
- **Бросается выполнять задачу первым знакомым способом**, не анализируя альтернативы
- **Игнорирует собственные правила** — читает CLAUDE.md и тут же забывает 80%

## Решение: СТОП-протокол + три уровня памяти

### СТОП-протокол (главное нововведение v2.0)

Принудительный чеклист из 10 пунктов ВХОДА и 4 пунктов ВЫХОДА. Claude **не имеет права начинать работу** пока не выполнит чеклист. Не имеет права считать задачу завершённой пока не выполнит ВЫХОД.

**Ключевое правило:** Скилл = ЧТО делать. Протокол = КАК делать. **Протокол первичен.** При запуске любого скилла (`/report`, `/hh` и т.д.) сначала выполняется протокол, потом скилл.

```
ЧЕКЛИСТ ВХОДА                          ЧЕКЛИСТ ВЫХОДА
□ Прочитал GOAL.md                     □ Спросил "Доволен результатом?"
□ Прочитал CONTEXT.md                  □ GOAL.md → ARCHIVE.md
□ Прочитал MEMORY.md + память          □ Обновил log.md
□ Нашёл последний log.md              □ Извлёк навыки → skills/
□ Уточнил если неясно
□ WebSearch — анализ подходов
□ Показал план пользователю
□ Получил "ок"
□ Записал цель в GOAL.md
□ Создал журнал
```

### Три уровня памяти

```
┌─────────────────────────────────────────────────────┐
│  Уровень 1: Журнал задачи (journals/*/log.md)       │
│  Живёт: пока задача актуальна                       │
│  Создаётся: каждую сессию автоматически              │
├─────────────────────────────────────────────────────┤
│  Уровень 2: Навыки (~/.claude/skills/)              │
│  Живёт: постоянно                                   │
│  Создаётся: когда найден рабочий приём              │
├─────────────────────────────────────────────────────┤
│  Уровень 3: Память (memory/*.md + MEMORY.md)        │
│  Живёт: постоянно, загружается каждую сессию        │
│  Создаётся: при коррекциях, новых фактах, паттернах │
└─────────────────────────────────────────────────────┘
```

### GOAL.md + ARCHIVE.md — целеполагание

- **GOAL.md** — текущая активная цель. Записывается перед началом работы, перечитывается каждые 5 шагов. Когда нет активной задачи — пуст.
- **ARCHIVE.md** — история завершённых целей. Каждая завершённая цель переносится из GOAL.md сюда с датой и результатом.

## Установка

### Быстрая (через Claude Code)

Если у вас есть Claude Code с системой скиллов:
```
/setup
```

### Ручная

#### 1. Скопируй CLAUDE.md

```bash
# Глобальные правила (для всех проектов)
cp CLAUDE.global.md ~/.claude/CLAUDE.md

# Правила проекта (в корень рабочей директории)
cp CLAUDE.project.md /path/to/your/project/CLAUDE.md
```

#### 2. Создай структуру проекта

```bash
cd /path/to/your/project
mkdir -p journals
touch GOAL.md
touch ARCHIVE.md
```

#### 3. Инициализируй память

```bash
# Claude Code создаст memory/ автоматически при первом сохранении
# Но можно создать MEMORY.md заранее:
mkdir -p ~/.claude/projects/-path-to-project/memory
echo "# Memory" > ~/.claude/projects/-path-to-project/memory/MEMORY.md
```

> **Путь memory/**: Claude Code хранит память по проекту в `~/.claude/projects/<encoded-path>/memory/`. Путь кодируется заменой `/` на `-`.

## Как это работает на практике

### Сессия с протоколом

```
Пользователь: /report-mfo ЛОКО 1-7 апреля

Claude (ВХОД):
1. Читает GOAL.md — пуст, нет активной цели ✓
2. Читает CONTEXT.md проекта report/ ✓
3. Читает MEMORY.md ✓
4. Находит последний log.md по теме report ✓
5. — (всё ясно из команды)
6. Анализ подходов: SQL через insapp-db MCP (прямой доступ) ✓
7. Показывает план: "Цель: отчёт ЛОКО за 1-7 апреля. Подход: SQL. План: 1... 2... 3..."
8. Ждёт "ок" от пользователя
9. Записывает цель в GOAL.md
10. Создаёт journals/2026-04-10-report-loko/log.md

Пользователь: ок

Claude (РАБОТА): выполняет план по шагам...

Claude (ВЫХОД):
1. "Доволен результатом?"
2. Да → переносит GOAL.md → ARCHIVE.md
3. Обновляет log.md (статус: завершено)
4. Сохраняет навыки если были
```

### Без протокола (как было раньше — НЕПРАВИЛЬНО)

```
Пользователь: /report-mfo ЛОКО 1-7 апреля

Claude: *сразу лезет в SQL, делает запросы, выдаёт результат*
- Не прочитал контекст
- Не проверил предыдущие сессии
- Не показал план
- Не спросил доволен ли пользователь
- Не записал в GOAL.md / ARCHIVE.md
- Навыки не извлечены
```

```
            ├── project_*.md     # Факты о проекте
            └── user_*.md        # Профиль пользователя

```

## Ключевые принципы

### 1. Протокол первичен над скиллами
Скилл описывает предметную область. Протокол описывает порядок работы. При конфликте — протокол побеждает.

### 2. Анализ подходов перед стартом
WebSearch обязателен перед составлением плана. Один раз в начале + повторно при тупняке. 5 минут анализа экономят часы работы.

### 3. GOAL.md — якорь целеполагания
Цель записывается ДО начала работы. Перечитывается каждые 5 шагов. Переносится в ARCHIVE.md после завершения. Без GOAL.md Claude теряет фокус.

### 4. Навыки создаются СРАЗУ
Не жди конца сессии. Нашёл рабочий приём → сразу в `~/.claude/skills/`. Сессия может оборваться в любой момент.

### 5. "Доволен результатом?" — обязательно
Claude не имеет права считать задачу завершённой без подтверждения пользователя. Даже если технически всё сделано.

### 6. Неполный результат = провал
50% транскрипции, 3 из 10 файлов — это НЕ успех. Минимум 3 разных подхода, потом СТОП и доклад пользователю.

## FAQ

**Q: Не замедлит ли протокол работу?**
Чеклист ВХОДА занимает ~30 секунд (чтение файлов). Анализ подходов — 1-2 минуты. Зато Claude не потратит 30 минут на кривой путь.

**Q: Нужен ли WebSearch для рутинных задач?**
Да, но он может быть быстрым. Для знакомой задачи — достаточно убедиться что подход по-прежнему оптимален. Для новой — полноценный анализ.

**Q: Что если Claude всё равно игнорирует протокол?**
СТОП-протокол стоит в самом начале CLAUDE.md — это первое что видит Claude. Формулировки императивные ("НЕ НАЧИНАТЬ", "НЕ ПОКАЗЫВАТЬ план"). Если всё равно игнорирует — добавьте hooks для enforcement.

**Q: Можно ли использовать с Cursor / другими IDE?**
Да. CLAUDE.md и skills/ работают в любом окружении где запускается Claude Code.
